# まちるだいすけと めむろたんけん

芽室町について、小学校3・4年生が1回5問ずつ学ぶ静的クイズです。  
公開中だった75問・3コース・出典情報とキャラクターを引き継ぎ、短時間で再挑戦しやすい形に改善しています。

## 特長

- 1回5問。25問の問題バンクから、前回と違う問題を優先して出題
- 正解・不正解の直後に、前向きな言葉と解説を表示
- 選択肢の順番を毎回シャッフル
- 間違えた問題だけをすぐに復習
- バッジ数・累計正解数・コース別最高点を端末内に保存
- 問題と選択肢の読み上げ（Chromebookの音声機能を使用）
- キーボード、タッチ、スクリーンリーダーに配慮
- 一度読み込んだ後は、対応ブラウザでオフライン利用可能
- 個人情報、ログイン、外部API、APIキーは不使用

## フォルダ構成

```text
memuro/
├─ index.html                 画面の骨組み
├─ manifest.webmanifest       ホーム画面追加用の設定
├─ sw.js                      オフライン用キャッシュ
├─ css/
│  └─ styles.css              見た目とレスポンシブ設定
├─ js/
│  └─ app.js                  出題・採点・記録・読み上げ
├─ data/
│  └─ questions.json          コースと全問題（先生が主に編集）
├─ images/
│  ├─ machiru-front.png
│  └─ machiru-wave.png
└─ IMPROVEMENTS.md            改善前後と変更理由
```

普段の問題追加・修正では、原則として `data/questions.json` だけを編集します。

## 問題を追加する

### 1. 追加先のコースを決める

`course` は次のどれかです。

- `beginner`：まちたんけん
- `intermediate`：くらし・しごとたんけん
- `advanced`：れきし・文化たんけん

### 2. `questions` の末尾に1問追加する

直前の問題の末尾にカンマが必要です。最後の問題の末尾にはカンマを付けません。

```json
{
  "id": "b26",
  "course": "beginner",
  "question": "芽室町で有名な作物は？",
  "choices": ["スイートコーン", "パイナップル", "バナナ"],
  "answer": 0,
  "explanation": "芽室町では、スイートコーンなどの畑作がさかんです。",
  "source": "めむろ学 p.26",
  "image": null
}
```

| 項目 | 書き方 |
|---|---|
| `id` | 全問題で重ならない短いID。既存IDは変更しない |
| `course` | 上記3コースのいずれか |
| `question` | 1文で短く。小学3・4年生が読める表現 |
| `choices` | 必ず3つ。複数正解にならないようにする |
| `answer` | 正解の位置。最初は `0`、2番目は `1`、3番目は `2` |
| `explanation` | 「なぜか」が分かる肯定的な説明 |
| `source` | 教材名・ページ・公式資料名 |
| `image` | 将来の画像用メモ。使わない場合は `null` |

### 3. 公開前に先生用チェックを開く

公開URLの末尾に `?teacher=1` を付けます。

```text
https://ユーザー名.github.io/リポジトリ名/?teacher=1
```

コースごとの問題数と、出典不足などの警告が表示されます。ID重複、選択肢数、正解番号などの重大な誤りは、通常画面でも読み込み時に検出します。

### 問題作成の確認項目

- 問い方は短く、1問で1つだけ尋ねているか
- 正解は資料で確認できるか
- 誤答は「ありそうだが明確に違う」内容か
- 正解の位置が毎回同じでも問題ない（表示時に自動シャッフルされる）
- 間違えた子が傷つかない解説になっているか
- 固有名詞、アイヌ文化、歴史的事項は出典を再確認したか

## 日常の運用

### 授業で使う

1. 児童はコースを1つ選ぶ
2. 5問に回答する（約3〜5分）
3. 結果画面で正解語を振り返る
4. 間違い復習、または「もう1回」に進む

読みが難しい児童は、各問題の「よむ」ボタンを使えます。効果音は初期状態でOFFです。

### 記録について

記録はブラウザの Local Storage にだけ保存されます。氏名は入力しません。

- 同じChromebook・同じChromeプロフィールでは継続
- 別の端末には同期しない
- ブラウザデータを消すと記録も消える
- 先生側に児童別データは送信されない

## GitHub Pagesで公開する

1. このフォルダの中身をGitHubリポジトリへ置く
2. GitHubのリポジトリで `Settings` → `Pages` を開く
3. `Deploy from a branch` を選ぶ
4. 公開するブランチとフォルダを選び、保存する
5. 表示された公開URLを開く

サーバー、データベース、ビルド作業は不要です。

更新後に古い表示が残る場合は、ページを再読み込みしてください。オフラインキャッシュを大きく変更したときは、`sw.js` の `CACHE_NAME`（例：`v2`）を `v3` に変更します。

## 手元で確認する

JSONはブラウザの安全制限により、`index.html` のダブルクリックでは読み込めません。次のどちらかで確認します。

- GitHub Pagesの公開URLで確認する
- VS Codeの「Live Server」など、学校で許可されたローカルプレビュー機能を使う

確認する画面幅の目安：

- Chromebook：1366 × 768
- 小型Chromebook：1024 × 600
- タブレット：768 × 1024
- スマートフォン：375 × 667

## 公開前チェック

- 3コースが表示される
- 各コースで5問が出る
- 選択肢の順番が再挑戦時に変わる
- 正解・不正解の表示と解説が合っている
- 「つぎへ」「やめる」「もう1回」「まちがい復習」が動く
- 途中でトップへ戻り、「つづける」で再開できる
- 再読み込み後もバッジと累計正解が残る
- Tabキーだけで全操作ができる
- 画面拡大200%でも横にはみ出さない
- コンソールにJavaScriptエラーがない

## カスタマイズ

- 色・余白・文字サイズ：`css/styles.css`
- 1回の問題数：`data/questions.json` の `sessionSize`
- コース名・説明・絵文字：`data/questions.json` の `courses`
- 出題・採点・保存：`js/app.js`

大きな機能を足す前に、「5問で学び、すぐ振り返る」流れを壊さないか確認してください。
